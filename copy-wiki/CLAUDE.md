# 미도리작업실 카피 파이프라인 위키 — 운영 규칙

이 저장소는 일반 지식 위키가 아니다. **내 목소리로 포트폴리오 카피를 쓰기 위한 재료 창고**다.

## 정체성

- 한 줄 정의: {{여기를 직접 채워주세요 — 예: 만드는 사람이자 기록하는 사람}}
- 역할: {{여기를 직접 채워주세요 — 예: 미도리작업실(meedorri) 운영, 제품 디자인·포토그래피}}
- 판단 기준: {{여기를 직접 채워주세요 — 예: "귀여워서"가 아니라 왜 오래 기억되는가}}

> 이 섹션은 본인이 직접 쓰는 것을 권장합니다. Claude가 추측해서 채우지 않습니다.

## 대원칙

raw는 목소리(How), wiki는 관점(What).

- `raw/` — 원본. Claude는 **읽기만** 한다. few-shot은 오직 여기서만 가져온다.
- `wiki/` — Claude가 정리한 판단 기준. **문체 모방 금지** — 그건 Claude의 톤이지 내 톤이 아니다.

raw/와 wiki/ 내부는 플랫하게 유지한다 (하위 폴더 금지). 분류는 `wiki/index.md`에서만 한다.

## 폴더 구조

### raw/ — 사용자가 채워 넣는다. Claude는 읽기 전용.

| 파일 | 내용 |
|---|---|
| `voice-portfolio.md` | 이미 발행한 포트폴리오·케이스스터디 본문 |
| `voice-personal.md` | 인스타 캡션, 에세이, 사적인 글 |
| `project-{이름}.md` | 개별 프로젝트 브리프·이미지목록·회고 |
| `references.md` | 내가 좋아하는 남의 글 모음 — **내 목소리 학습에는 사용 금지** |

### wiki/ — Claude가 작성하고 관리한다.

| 파일 | 내용 |
|---|---|
| `index.md` | 카테고리별 목차. 페이지 추가 시마다 갱신 |
| `voice-rules.md` | 어휘·문체·특수 타이포 지문 문서 |
| `log.md` | 작업 이력 |
| `project-{이름}.md` | 정리된 프로젝트 페이지 |

## 5가지 모드

**[1] 목소리 학습** — `raw/voice-*.md`에 새 글 추가됨
→ 어휘·패턴·시그니처 문장을 관찰해 `voice-rules.md` 업데이트

**[2] 프로젝트 등록** — `raw/project-*.md` 추가됨
→ `wiki/project-{이름}.md` 생성. 맨 위에 `> ` callout으로 "이 프로젝트가 왜 오래 기억되는가" Takeaway 필수 + `index.md` 등록

**[3] 카피 생성** — 프로젝트명 + 목적(About/Caption/Slogan) 지정받음
→ `voice-rules.md` + 해당 `wiki/project-{이름}.md` + `raw/` few-shot 3개를 조립
→ 카피 3안 생성 → 검증(금지 어휘·룰 위반 체크) 후 반환

**[4] 건강검진**
→ 보이스 드리프트(톤 이탈·금지 어휘·필수 어휘 자연 등장) 리포트
→ 위키 구조(깨진 링크·index 누락·raw/wiki 불일치) 리포트

**[5] 질문하기**
→ 위키에서 답하고, 근거가 되는 raw 원문을 인용한다.
→ 위키에 없으면 "없다"고 말한다 — 추측으로 채우지 않는다.

## 절대 규칙

1. `voice-rules.md`는 raw 원본에서 관찰한 패턴만 기록한다 — 모든 규칙에 근거 예문 필수. 추측·클리셰 금지.
2. 금지 어휘: 트렌디한 / 세련된 / 감각적인 / 새롭게 재해석한 / 차별화된 + 근거 없는 형용사 일체.
3. `raw/`는 절대 수정하지 않는다.
4. 모든 작업을 `wiki/log.md`에 날짜와 함께 기록한다. 구조 변경은 이유도 함께 남긴다.
